## 4. Objective

### 4.1 핵심 목표

- 노후화된 홈페이지 UI를 개선한다.
- 태일씨앤티의 회사 정보, 사업 실적, ESG 경영, 채용 정보를 명확하게 전달한다.
- 외부 인사와 채용 지원자가 쉽게 탐색할 수 있는 반응형 웹사이트를 만든다.

### 4.2 성공 기준

- PC, 태블릿, 모바일에서 화면이 깨지지 않는다.
- 정적 호스팅 가능한 구조로 배포할 수 있다.
- 주요 메뉴 구조가 빠짐없이 구현된다.
- 이미지/영상 최적화로 초기 로딩 성능을 확보한다.
- 중간 발표 시 `node_modules` 제외 소스 기준으로 `npm run dev` 실행이 가능해야 한다.

### 7.4 개발 요구 사항

1. 영상과 이미지는 `WebM`, `WebP` 등으로 최적화해 로딩 속도를 확보한다.
2. 모바일, 태블릿, PC 어디서든 화면이 깨지지 않는 반응형 웹으로 구현한다.
3. 서버 사이드 로직이 없는 정적 웹사이트 구조로 설계한다.
4. 운영은 태일씨앤티가 직접 관할하므로 정적 호스팅 환경을 고려해 프로젝트를 개발한다.
5. 중간 발표는 빌드 전 결과물로 확인하며, 제출 시 `node_modules`는 제외한 상태에서 `npm run dev`로 실행 가능해야 한다.
6. 절대 경로 대신 상대 경로를 권장하며, 모든 리소스는 폴더 내에 포함한다.
7. 기존 태일씨앤티 홈페이지의 사진과 글을 참고해 작성할 수 있다.
8. 최종 제출물은 아래 항목을 포함한다.
   - 빌드 결과물
   - 소스 코드 전체
   - 기술서

### 7.5 개발/운영 제약

- JavaScript 기반으로 개발한다.
- 구동 환경은 Node.js 22 LTS를 기준으로 한다.
- 개발 중 확인 방식은 `npm run dev` 기준이어야 한다.
- 제출 소스는 `node_modules` 없이 전달 가능한 구조여야 한다.
- 배포 환경은 정적 호스팅을 전제로 한다.
- 경로 의존성 문제를 피하기 위해 상대 경로 중심으로 구성한다.
